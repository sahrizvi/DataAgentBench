HINTS:
- The same canonical award_id appears under a different surface form in each database. Each table independently applies: (1) OCR-like character substitutions (0↔O, 1↔I, 5↔S, 2↔Z, 6↔G, 8↔B) on a per-character basis, (2) one of 207 prefix variants, (3) upper/lower/original case, (4) separator characters at letter↔digit boundaries. Cross-table joins cannot be done by string equality or a simple regex — they require entity resolution (loading all award_ids from both tables and fuzzy-matching).
- Similarly, the same canonical recipient UEI appears under a different surface form in contracts_db vs recipients_db, with the same OCR + prefix + case + separator pipeline applied independently per table.
- naics_code in contracts also undergoes OCR substitutions plus format variants (e.g. "336411", "naics-33G411", "33-G411"). naics_sectors in agencies_db stores canonical codes.
- contracts_db.contracts.awarding_agency, awarding_sub_agency, and funding_agency may appear under multiple surface forms for the same canonical agency; agencies_db.agency_aliases resolves surface forms to a canonical agency name.
- recipients_db.recipients.name has multiple surface-form variants for the same canonical entity (corporate-suffix, case, and whitespace differences); some canonical entities are represented by more than one distinct UEI.
- recipients_db.recipients.state may appear under multiple surface forms (e.g. "CA", "California", "Calif.").
- contract_amounts.amount_text and recipients.total_amount_text encode dollar amounts in many different text formats: plain integers, comma-separated numbers, dollar-prefixed, USD-suffixed, scaled abbreviations (K / M / B), full English word form, and sum-of-parts expressions (e.g. "$1,050,000 + $450,000"). Every format encodes the exact amount and is unambiguously parseable.
- A small subset of awards (~3.5%) have a superseded amount entry in contract_amounts. These rows are identifiable because the award_id ends with the suffix _OLD (appended to the same surface-form as the primary entry). The superseded amount differs from the primary amount. A proper join from contracts to contract_amounts via normalized award_id will naturally skip these _OLD rows.
- A subset of contracts are missing the English description and only carry a non-English description.
