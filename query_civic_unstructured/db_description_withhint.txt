HINTS:
- Projects have two types: "capital" (Capital Improvement Projects for infrastructure, parks, roads) and "disaster" (Disaster Recovery Projects, often FEMA-funded or related to Woolsey Fire recovery).
- Projects have three statuses: "design" (in planning/design phase), "completed" (finished), and "not started" (identified but not begun).
- The topic field contains comma-separated keywords. Common topics include: "park", "road", "FEMA", "fire", "emergency warning", "drainage", "storm drain", "highway", "bridge", "playground", "water treatment", "guardrail".
- Dates (st, et fields) use flexible formats like "2022-Spring", "2022-Fall", "2022-02", "2022-March". Use substring matching (e.g., checking if "2022" is in the date string) for year-based queries. Spring = March-May, Summer = June-August, Fall = September-November.
- The Project_Name in the Funding SQLite table matches the project names that can be extracted from the civic documents in MongoDB.
- Disaster project names often include suffixes like "(FEMA Project)", "(CalJPIA Project)", or "(CalOES Project)".
